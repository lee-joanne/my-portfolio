Yakker

A Twitter-style social media platform.

Problem

In an increasingly noisy and identity-driven digital world, many users crave a space to share thoughts without the pressure of building a public persona. While major platforms like Twitter offer real-time interaction, they often come with complex algorithms, content visibility issues, and identity-driven engagement. I wanted to explore what it would look like to build a stripped-back, community-focused social media platform, one where users could post freely, follow others, and engage without the social pressure of being "seen." Yakker was born out of a personal challenge: could I design and build a working social platform from scratch, with a focus on anonymity and simplicity?

Solution

Yakker is a Twitter-style microblogging platform built using React on the front end and Django REST Framework on the back end. It allows users to sign up, follow others, and share short posts in a clean, anonymous feed. The design is intentionally minimal to prioritize content over identity. There are no email verifications or required personal details. Just text-based interactions that emphasize conversation. Building Yakker helped me dive deep into authentication, API development, and front-end state management, while also exploring the UX implications of designing for low-pressure social interaction.

Research

Kickoff

As a solo designer and developer for this project, I began by defining clear goals for Yakker: to create a lightweight, anonymous social media platform inspired by Twitter, but without the pressure of personal branding. I conducted a competitive audit of popular platforms, documenting early assumptions and identifying critical research questions to guide the build.

Competitive Analysis

I examined onboarding and interaction patterns across platforms like Twitter, Threads, and Reddit. A key insight was that users often drop off during lengthy sign-up flows. Yakker’s approach prioritizes minimal friction by allowing immediate posting after sign-up.

Target Users & Privacy

Yakker is aimed at users who want to post freely without being tied to their identity — including students, professionals, and casual users. I also considered the importance of limiting personal data collection, keeping privacy and safety top of mind.

Key Insights

  • Users are more likely to engage when onboarding is simple and non-invasive
  • Removing popularity metrics encourages authentic content
  • Anonymity boosts posting frequency but needs moderation
  • Transparency around data collection builds trust

Ideation

User Stories

For the user stories, I have split them up depending on the user action:

Authentication

Feature User Story
Log In As a user I can easily log in so that I can quickly post, add more comments, or delete content as I wish.
Log Out As a user I can easily log out so that I can have more security on my account.
Sign Up As a user I can sign up easily with just a username and password so that I have the ability to create posts or comments without having to share my email address.
Permissions As a user I cannot edit/delete posts/comments/likes that are not mine so that I can be assured that my posts/comments/likes are protected and can only be changed by myself.
Like Rules As a user I can only like/unlike other users' posts and comments so that I cannot tamper with the software and like my own posts/comments.
Follow Rules As a user I can only follow other users so that I cannot tamper with the software and follow myself.
Login Status As a user I can see whether I am logged in or not so that I understand my current authentication status.
Session Persistence As a user I can stay logged in so I don’t have to keep signing in unless I manually log out.

Posts

Feature User Story
Create As a user I can create new posts so that I can share images, my thoughts, or what I'm doing.
Edit As a user I can edit posts so that I can update them if I change my mind.
Delete As a user I can delete my posts so that I can remove content I no longer want to share.
Details As a user I can view post details including time, creator, and comments.
Like As a user I can like posts to show appreciation.
Unlike As a user I can remove likes if I change my mind.

Comments

Feature User Story
Create As a user I can post a comment on a post to contribute discussion.
Delete As a user I can delete my comments on a post to remove them as I wish.
Read As a user I can read comments on a post to see what others are saying.
Edit As a user I can edit my comments to refine them later.
Like As a user I can like comments to show appreciation.
Unlike As a user I can unlike comments if I change my mind.

Navigation

Feature User Story
Nav Bar As a user I can see the navigation bar at the top of every page so that I can easily navigate.
Page Speed As a user I can navigate between pages quickly without full reloads.
404 Page As a user I can see a styled 404 page if I land on an invalid URL.
500 Page As a user I can see a consistent 500 error page for backend issues.
Inf. Scroll As a user I can infinitely scroll down to see more posts and comments without clicking extra links.
Conditional Nav As a user I see login/signup links when logged out, and profile/post links when logged in.

Profile

Feature User Story
Avatars As a user I can view user avatars to visually identify them.
User Profiles As a user I can view detailed pages for users showing their posts and stats.
Follow/Unfollow As a user I can follow or unfollow users to customize my feed.
Popular Users As a user I can see the most followed profiles to find trending accounts.

General

Feature User Story
Search As a user I can search for posts or users to quickly find what I'm looking for.
About Page As a user I can easily find the 'About' link in the nav to learn about the platform’s purpose.

Design

Wireframes

In the early stages of the design process, I adopted a mobile-first approach by sketching out key screens on paper. This allowed me to quickly visualize core user flows and focus on content hierarchy, clarity, and essential interactions. After solidifying the basic structure, I translated these sketches into digital low-fidelity wireframes using Whimsical. These wireframes helped define layout and user flow while keeping the visuals intentionally minimal. Since the primary goal of this project was to deepen my development skills, I chose to move directly into development after the low-fidelity stage. I used the wireframes as a functional blueprint to guide the front-end build and ensure that the app’s key features, posting, following users, and engaging anonymously, were implemented with a clear and consistent structure. While I didn’t create high-fidelity mockups, I continuously refined UI components during development to balance usability with simplicity.

Paper Wireframes

I began with hand-drawn wireframes to quickly explore layout and flow options without committing to visuals. This helped me rapidly ideate and sketch out key screens such as the homepage, user feed, and post interaction views.

Low-Fidelity Wireframes: Mobile

After validating core flows, I moved into digital low-fidelity wireframes using Whimsical. I quickly visualized the layout of the social feed, creating posts, commenting on other user's posts, and the user's profile page, ensuring all user needs are met.

Low-Fidelity Wireframes: Desktop

To refine the layout for desktop screens, I transitioned into the digital wireframes for desktop, introducing more accurate sizing and key UI components. This phase focused on improving clarity in post layout, profile summaries, and feed structure.

Style Guide

The style guide for Yakker is designed to create a fun, modern, and readable user experience that reflects the platform’s casual and community-driven vibe. It features two primary fonts sourced from Google Fonts: Fredoka One, a playful and bold cursive used for headers, post titles, and navigation to establish a clear visual hierarchy; and Raleway, a clean and simple sans-serif chosen for body text and longer content to maximize readability.

Logo

The logo was created using Logo.com. The goal was to feature a fun, cartoon illustration of a yak to nicely tie the platform name and logo together.

Colour Palette

The color palette, inspired by a scheme developed by Logo.com, strategically uses a primary color for most text, a secondary color to highlight interactive elements like links and buttons on hover, and an accent color to emphasize key details such as usernames and active navigation links. Together, these choices ensure a cohesive and approachable aesthetic that supports Yakker’s goal of fostering open, low-pressure social interaction.

Final Product

Although mockups are no longer available due to deployment expiry, I captured key screenshots of the final built product before it went offline. These images showcase the core interface, including the responsive layout, and essential user interactions such as posting, following, and profile viewing. The final design reflects the original goals of the platform, a clean, minimal, and anonymous social media experience, with thoughtful UI choices like soft tones, clear typography, and intuitive flows. As both the designer and developer, I translated my low-fidelity wireframes directly into code, focusing on responsiveness, usability, and aligning functionality with user stories defined early in the process.

Reflection

Working on this project was a genuinely enjoyable experience. I appreciated the opportunity to take ownership of both the design and development process, which allowed me to fully immerse myself in building the product end-to-end. However, working solo also highlighted how valuable it would be to collaborate within a larger team environment. Having dedicated designers and developers to brainstorm with, exchange ideas, and iterate collectively would bring fresh perspectives and help push the project even further. I am especially eager to work in a more Agile setup, where cross-functional collaboration and continuous feedback drive the process.

On the design side, I see great value in investing more time into creating high-fidelity wireframes and prototypes before diving deep into development. This would help double-check that all user needs are met early on, reducing the risk of costly changes later and ensuring a smoother development workflow. Overall, this project has strengthened my passion for building meaningful products, and I look forward to contributing in team-based environments where I can learn from others and help deliver even better solutions.
